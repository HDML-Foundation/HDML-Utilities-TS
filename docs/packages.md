# Packages — public API per workspace

> **Scope:** Per-package public surface (exports, key types, runtime behaviors). Open this
> when you need to know what a specific `@hdml/*` package offers without reading its source.
> For how packages depend on each other and the end-to-end pipeline, see [architecture.md](architecture.md).
> For wire formats, see [schemas.md](schemas.md).

All packages publish at the **same version** (currently `0.0.2-alpha.13`). All emit CJS +
ESM + `.d.ts` + minified IIFE. Each runtime-bearing `index.ts` also registers itself on
`globalThis["@hdml/<name>"]` as a side-effect — see [architecture.md#globalthis-side-effects](architecture.md#globalthis-side-effects-every-entry-point).

## @hdml/schemas

**Purpose:** TypeScript FlatBuffers bindings — the wire-format contract.
**Source:** Generated by `flatc --ts` from [HDML-Schemas/src/*.fbs](../HDML-Schemas/) at
build time. Output lands in `packages/schemas/src/document/` and `packages/schemas/src/enum/`,
both gitignored.
**Entry point:** [packages/schemas/src/index.ts](../packages/schemas/src/index.ts).

Exports two groups of names — **enums** and **structs** — re-exported from generated files.

Enums: `AggregationTypeEnum`, `ConnectorTypesEnum`, `DataTypeEnum`, `DateUnitEnum`,
`DecimalBitWidthEnum`, `FilterNameEnum`, `FilterOperatorEnum`, `FilterTypeEnum`,
`JoinTypeEnum`, `OrderTypeEnum`, `TableTypeEnum`, `TimeUnitEnum`, `TimeZoneEnum`.

Structs: `HDOMStruct`, `IncludeStruct`, `ConnectionStruct`, `ConnectionOptionsStruct`,
`ConnectionParametersStruct`, per-connector parameter structs (`BigQueryParametersStruct`,
`JDBCParametersStruct`, `GoogleSheetsParametersStruct`, `ElasticsearchParametersStruct`,
`MongoDBParametersStruct`, `SnowflakeParametersStruct`), `ModelStruct`, `TableStruct`,
`JoinStruct`, `FrameStruct`, `FieldStruct`, `FieldTypeStruct`, type-options structs
(`CommonParametersStruct`, `DateParametersStruct`, `DecimalParametersStruct`,
`TimeParametersStruct`, `TimestampParametersStruct`, `DataTypeOptionsStruct`,
`UnspecifiedParametersStruct`, `ExpressionParametersStruct`), filter structs
(`FilterStruct`, `FilterClauseStruct`, `FilterOptionsStruct`, `NamedParametersStruct`,
`KeysParametersStruct`), and the document-files trio: `FileStruct`, `DocumentFilesStruct`,
`DocumentFileStatusesStruct`.

**Build hook:** `npm run compile_fbs` regenerates from `.fbs`. The `build` script runs it
automatically before lint.

**Dep:** `flatbuffers` (24.3.25 — must match the `flatc` version).

## @hdml/types

**Purpose:** Hand-authored TypeScript interfaces and lookup tables that mirror the FlatBuffers
schema, plus the HDML tag / attribute / enum-value name constants used by the parser and
stringifier.

**Entry point:** [packages/types/src/index.ts](../packages/types/src/index.ts).

**Object interfaces:** `HDOM`, `Include`, `Connection`, `Model`, `Table`, `Join`, `Frame`,
`Field`, `FieldType`, `FilterClause`, `Filter`, and per-connector / per-parameter parameter
types (`BigQueryParameters`, `JDBCParameters`, `GoogleSheetsParameters`,
`ElasticsearchParameters`, `MongoDBParameters`, `SnowflakeParameters`, `ConnectionOptions`,
`CommonParameters`, `DateParameters`, `DecimalParameters`, `TimeParameters`,
`TimestampParameters`, `ExpressionParameters`, `NamedParameters`, `KeysParameters`).

**Lookup tables** (string constants — used by parser to read attributes and by stringifier to
emit them):

- `HDML_TAG_NAMES` — `hdml-include`, `hdml-connection`, `hdml-model`, `hdml-table`,
  `hdml-join`, `hdml-frame`, `hdml-field`, `hdml-connective`, `hdml-filter`,
  `hdml-filter-by`, `hdml-group-by`, `hdml-split-by`, `hdml-sort-by`.
- `*_ATTRS_LIST` — `CONN_ATTRS_LIST`, `MODEL_ATTRS_LIST`, `TABLE_ATTRS_LIST`,
  `JOIN_ATTRS_LIST`, `FRAME_ATTRS_LIST`, `FIELD_ATTRS_LIST`, `FILTER_ATTRS_LIST`,
  `CONNECTIVE_ATTRS_LIST`, `INCLUDE_ATTRS_LIST`.
- `*_VALUES` — `AGGREGATION_VALUES`, `BITWIDTH_VALUES`, `CONN_TYPE_VALUES`,
  `CONNECTIVE_OP_VALUES`, `DT_UNIT_VALUES`, `FIELD_TYPE_VALUES`, `FILTER_NAME_VALUES`,
  `FILTER_TYPE_VALUES`, `JOIN_TYPE_VALUES`, `ORDER_VALUES`, `TABLE_TYPE_VALUES`,
  `TIMEZONE_VALUES`.

**No runtime side-effects.** Pure types + constants. `test` script is intentionally empty.

**Deps:** `@hdml/schemas` (for the enum types), `flatbuffers`.

## @hdml/buffer

**Purpose:** FlatBuffers (de)serialization for `HDOM` and its children, plus the
`DocumentFilesStruct` packaging used by HDIO's compile pipeline.

**Entry point:** [packages/buffer/src/index.ts](../packages/buffer/src/index.ts).

**High-level exports** (all switch on `StructType`):

| Symbol | Signature | Defined in |
|---|---|---|
| `serialize(data, type = HDOMStruct)` | `(HDOM \| Connection \| Model \| Frame \| FileStatuses, StructType) → Uint8Array` | [packages/buffer/src/serialize.ts:44](../packages/buffer/src/serialize.ts#L44) |
| `deserialize(bytes, type = HDOMStruct)` | `(Uint8Array, StructType) → HDOM \| Connection \| Model \| Frame \| FileStatuses` | [packages/buffer/src/deserialize.ts:50](../packages/buffer/src/deserialize.ts#L50) |
| `structurize(bytes, type = HDOMStruct)` | `(Uint8Array, StructType) → HDOMStruct \| ConnectionStruct \| ModelStruct \| FrameStruct \| DocumentFileStatusesStruct` | [packages/buffer/src/structurize.ts:49](../packages/buffer/src/structurize.ts#L49) |
| `fileifize(hdom: HDOM): Uint8Array` | Packs HDOM as a `DocumentFilesStruct` with per-connection / per-model / per-frame `FileStruct` entries | [packages/buffer/src/fileifize.ts:40](../packages/buffer/src/fileifize.ts#L40) |
| `StructType` (enum) | `HDOMStruct=1, ConnectionStruct=2, ModelStruct=3, FrameStruct=4, FileStatusesStruct=5` | [packages/buffer/src/StructType.ts](../packages/buffer/src/StructType.ts) |
| `FileStatuses` (type) | `{ names: string[]; statuses: string[]; messages: string[] }` (parallel arrays) | [packages/buffer/src/FileStatuses.ts](../packages/buffer/src/FileStatuses.ts) |

**Low-level converters** — not re-exported from the package index (use the high-level
functions). They live under [packages/buffer/src/bufferify/](../packages/buffer/src/bufferify/)
(TS → FlatBuffers offsets) and [packages/buffer/src/objectify/](../packages/buffer/src/objectify/)
(FlatBuffers structs → TS). One pair per node type: `HDOM`, `Connection`, `Model`, `Frame`,
`Field`, `FilterClause`. The `bufferify/` side additionally has `bufferifyDocumentFiles`,
`bufferifyFile`, `bufferifyFileStatuses`; `objectify/` has `objectifyFileStatuses` (no
matching `objectifyDocumentFiles` — `fileifize` is one-way; the consumer splits and
re-`structurize`s on the read side).

**Deps:** `@hdml/types`, `@hdml/schemas`, `flatbuffers`.

## @hdml/stringifier

**Purpose:** Generate SQL or HTML strings from FlatBuffers structs. **Inputs are structs, not
TS interfaces** — bufferify first if you only have a TS object.

**Entry point:** [packages/stringifier/src/index.ts](../packages/stringifier/src/index.ts).

| Symbol | Signature | Defined in |
|---|---|---|
| `getConnectionSQLs(conn: ConnectionStruct): string[]` | Emits `show catalogs like …`, `drop catalog …`, `create catalog … using <connector> with (…)` triple. Dispatches on `ConnectorTypesEnum`. | [packages/stringifier/src/connection.ts:25](../packages/stringifier/src/connection.ts#L25) |
| `getConnectionHTML(conn: ConnectionStruct): string` | HDML `<hdml-connection …>` element. | [packages/stringifier/src/connection.ts:254](../packages/stringifier/src/connection.ts#L254) |
| `getModelSQL(model: ModelStruct, level?: number): string` | `with … select … from / join …`. Uses `sortJoins` for stable join order. | [packages/stringifier/src/model.ts:24](../packages/stringifier/src/model.ts#L24) |
| `getModelHTML(model: ModelStruct, level?: number): string` | HDML `<hdml-model>` element. | same file |
| `getFrameSQL(frame: FrameStruct, from: {name, sql}, level?: number): string` | `with "<src>" as (…) select … from "<src>" [where] [group by] [order by] offset N limit N`. Caller threads upstream SQL via `from`. | [packages/stringifier/src/frame.ts:17](../packages/stringifier/src/frame.ts#L17) |
| `getFrameHTML(frame: FrameStruct, level?: number): string` | HDML `<hdml-frame>` element including children. | [packages/stringifier/src/frame.ts:105](../packages/stringifier/src/frame.ts#L105) |

Indentation comes from a single 2-space constant `t` in [packages/stringifier/src/constants.ts](../packages/stringifier/src/constants.ts).

**Multi-table model field aliasing.** When a `ModelStruct` carries multiple tables (joined
or not), `getModelSQL` emits the outer `SELECT` with every field aliased as
**`"{table-name}_{field-name}"`** — see
[packages/stringifier/src/model.ts:51-63](../packages/stringifier/src/model.ts#L51-L63).
Downstream consumers (a `getFrameSQL` call sourcing this model, or an HDML author writing
an `hdml-frame` with `source="?hdml-model=…"`) must reference parent fields by that
compound name — bare field names won't resolve when several tables carry the same column. The rule
applies even to single-table models. Frame-to-frame projection in `getFrameSQL` performs
no such rewriting: parent-frame fields are exposed by their declared `name`.

**Deps:** `@hdml/schemas`, `@hdml/types` (dev: `@hdml/buffer` for tests).

## @hdml/buffer-adjacent: @hdml/hooks

**Purpose:** WASI/Javy I/O helpers — read/write JSON, strings, and `Uint8Array`s over
stdin/stdout/stderr inside `hdio.wasm`.

**Entry point:** [packages/hooks/src/index.ts](../packages/hooks/src/index.ts).

| Symbol | Reads from / writes to | Notes |
|---|---|---|
| `readUint8Array(): Uint8Array` | stdin (fd 0) | 1 KiB chunks until `readSync` returns 0. [packages/hooks/src/io/readUint8Array.ts](../packages/hooks/src/io/readUint8Array.ts) |
| `readString(): string` | stdin | TextDecoder over `readUint8Array`. |
| `readJson<T = unknown>(): T` | stdin | `JSON.parse(readString())`. |
| `writeUint8Array(bytes)` | stdout (fd 1) | — |
| `writeString(s)` | stdout | TextEncoder. |
| `writeJson(v)` | stdout | `JSON.stringify`. |

**Host contract:** The Go host (HDIO-Server) wires temp files on a tmpfs to fd 0/1, plus
fd 2 for stderr logs. See [integration.md#wasm-host-abi](integration.md#wasm-host-abi).

**Dep:** `@hdml/parser`, `@hdml/types`, `@hdml/schemas`. (The parser dep is for tests / a
future round-trip helper; the I/O primitives themselves do not touch HDML.)

## @hdml/common

**Purpose:** Shared third-party re-exports — the same `apache-arrow`, `uuid`, and
`throttle-debounce` instances every other package and every host consume. Avoids duplicate
copies in the Javy bundle.

**Entry point:** [packages/common/src/index.ts](../packages/common/src/index.ts).

Exports `arrow`, `uuid`, `throdeb` namespaces; also writes them to
`globalThis["@hdml/common"]`.

**Deps:** `apache-arrow ^11.0.0`, `uuid ^10.0.0`, `throttle-debounce ^5.0.2`.

Not typically imported by end users — most packages pull only what they need from
`@hdml/common`. Apache Arrow is here because HDIO's query path returns Arrow streams to the
client; staging it in `@hdml/common` keeps the version pinned across hosts.
